数据与算法期末复习

发表于 2023-02-24 更新于 2023-04-10 分类于 抱佛脚 阅读次数：

二元关系

设R是集合M上的一个二元关系

自反性：任意a属于M，都有(a,a)属于R

反自反性：任意a属于M，都有(a,a)不属于R

对称性：任意(a,b)属于R，有(b,a)属于R

反对称性：任意(a,b)属于R，且a不等于b，则有(b,a)不属于R

传递性：如果(a,b)和(b,c)属于R，则(a,c)属于R

等价关系：满足自反性、对称性、传递性。（例如=关系）

偏序关系：满足自反性、反对称性、传递性。（例如\(\le\)关系）

拟序关系（严格偏序关系）：满足反自反性、反对称性、传递性。（例如<关系）

全序关系：是偏序关系，且任意a，b，不是aRb就是bRa。（偏序关系包含全序关系）

队列

队列：队尾（rear）允许插入，队头（front）允许删除，插入操作成为入队，删除操作成为出队。

优先级队列：优先级队列的每次操作都是针对队列中具有最高优先权的元素。

  • 基于无序顺序表实现：入队\(O(1)\)，出队\(O(N)\)，访问最高优先级元素\(O(N)\)
  • 基于有序顺序表实现：入队\(O(N)\)，出队\(O(N)\)，访问最高优先级元素\(O(1)\)
  • 基于无序链表：入\(O(1)\)，出\(O(N)\)，访问最高优先级\(O(N)\)
  • 基于有序链表：入\(O(N)\)，出\(O(1)\)，访问最高优先级\(O(1)\)

根据元素在队列中的停留时间来认定优先权值，可以将栈和队列统一起来。

字符串

模式匹配：

KMP算法

首先对模式串P进行预处理，找出对于P\([0,j]\)的前缀和P\([1,j]\)的后缀最大相等的长度（设为next[j]），则在此处失配后，如果j>0，则匹配串指针j=next[j-1]，否则i++。全程主串的指针i不回溯。

Horspool算法

先将模式串和文本开头对齐，比对模式串最后一个字符，并依情况向后移动。最差效率\(O(nm)\)，对于随机文本来说则是\(O(n)\)。

Huffman树与编码

带权路径长度：

记为\(WPL=\Sigma W_iP_i\)，其中\(W_i\)为叶子结点的权值，\(P_i\)为叶子结点到根的路径长度。

Huffman树：

在所有路径长度相同的二叉树中，WPL最小的树称为HUFFMAN树（或最优二叉树）。

Huffman算法：

  1. n个权值构造n棵树，这些树只有根节点，没有左右子树。
  2. 选取集合中根值最小的两棵树，分别为左右子树构造一棵树，根节点权值设为左右子树根节点权值之和。
  3. 删去旧两棵树，加入新树。
  4. 循环往复直至只剩一棵树，这棵树就是Huffman树。

Huffman编码：

字符出现的概率即为Huffman树中叶子结点的权值，平均编码长度最小。（缺点：无法差错和纠错）

堆

堆是完全二叉树。

最大堆：堆中任一结点的值都小于其父结点的值，其堆顶元素最大。

最小堆：与上相反。

由于堆是完全二叉树，因此可以顺序储存在一维数组中，则A[i]的左孩子结点和右孩子结点分别为A[2i+1]和A[2i+2]。

自顶向下堆化：最大堆中某结点值小于子结点。O(h)

自底向上堆化：最大堆中某结点值大于父结点。O(h)

堆的插入操作即是在堆底加入点后，执行自底向上堆化，也可以据此构建新堆，这称为自顶向下的堆构造（有点弯弯绕绕的），复杂度小于O(nlogn)。

自底向上的堆构造复杂度为O(n)。

堆的删除堆顶操作：删除堆顶并返回值，将堆尾结点放到堆顶，并自顶向下堆化。

图

深度优先遍历

时间复杂度：对于邻接表，O(n+e)；对于邻接矩阵，O(n^2)

空间复杂度：对于邻接表和邻接矩阵都是O(n)

广度优先遍历

时间复杂度：对于邻接表，O(n+e)；对于邻接矩阵，O(n^2)

空间复杂度：O(e)或O(n)，根据具体实现决定

普里姆（Prim）算法：

  1. 初始化最小树为空集，从图中任取一个顶点加入最小树。
  2. 取出距离最小树最短的点和边，加入最小树。
  3. 重复直到最小树包含所有点。
  • 时间复杂度为O(n^2)，适合稠密图。

克鲁斯卡尔（Kruskal）算法：

  1. 对图中所有边按权值排序，初始化最小树为空。
  2. 取出权值最小的边，如果它在最小树中能构成环则舍弃，否则就加入最小树。
  3. 重复直到n-1条边加入了最小树。
  • 使用堆排序来对边进行排序。
  • 如果采用高级排序方法，时间复杂度为O(e*log2(e))，适合稀疏图。

迪克斯特拉（Dijkstra）算法：

求解单源最短路径

  1. 初始化最短路径树为空，取源点s。
  2. 沿s发出的边进行拓展，更新s到不在树中各顶点的最短距离。
  3. 找出距离s最近的不在树里的点v，将v和相应的边加入最短路径树。
  4. 沿v发出的边进行拓展，更新s到不在树中各顶点的最短距离。
  5. 重复直至最短路径树包含所有点。
  • 时间复杂度为O(n^2)，属于贪心算法。

弗洛伊德Floyd算法：

查找

平均查找长度\(ASL=\Sigma P_iC_i\)，后两个分别代表第i个元素被查找的概率和查找所需要的比较次数，例如顺序查找的ASL=(n+1)/2，属于蛮力算法。

折半查找：（线性查找表对于关键词来说是有序的）平均查找长度为O(log2(n))，属于减治算法。

斐波那契查找：添加关键词使表长为F(n)-1，则可以分为两个子表和一个中间点，表长分别为F(n-1)-1和F(n-2)-1。表长较大时平均性能要好于折半查找，最坏情况则不如。

哈希冲突处理：

开散列法

也成为链地址法，将散列到同一地址的不同关键词存入相应的链表中，一般选择平均链表长度为5~10.

闭散列法

也叫开放定址法，当表上已经有一个与待搜索关键词不同的元素，探测其他位置。

  • 在保证哈希表足够稀疏的情况下，线性探测散列具有最快的查找性能；
  • 双重散列使用内存最为有效，但时间复杂度高于线性探测散列，散列表趋近于满时，双重散列性能远优于线性探测散列。
  • 链地址法容易实现，但平均情况下性能不如开放地址法。

双重散列函数不能取0是为了防止死循环；又必须与表长互素，否则某些探测序列会非常短。

排序

简单排序

  • 插入排序算法时间复杂度与待排序序列的初始排列无关。

  • 希尔排序的性能决定于步长序列的选择，例如一个好的序列\(h_i=4^i+3*2^{i-1}+1\)

  • 选择排序比较次数多而交换次数少，适用于元素规模大但关键字规模小的情况。（不稳定）

  • 简单排序方法的平均时间复杂度都是O(n^2)

快速排序

  • 基于分治思想，最好情况下的时间复杂度为O(n*log2(n))，平均情况为O(nln(n))，最坏情况（这里是待排序列已经完全有序的情况）下退化到O(n^2)。

  • 不稳定。

  • 为避免最坏情况，可以采用中间元素法，即取序列左端元素、中间元素和右端元素，取关键字值位于中间的元素作为划分元素。

归并排序
  • 二路归并：时间复杂度O(n)，需要规模为n的空间来储存归并得到的新序列。利用它来实现下面的归并排序。
  • 自底向上的归并排序：时间复杂度为O(n*log2(n))，而且不会恶化。但需要与n成正比的额外内存空间。（稳定）
  • 自顶向下的归并排序：类似，但是从长到短归并，递归调用。
  • 归并排序顺序访问数据，因此可以作为链表排序选择的方法。
堆排序
  1. 对待待排序列，采用自底向上的堆构造方法来构造初始的最大堆。
  2. 堆尾和堆顶交换，堆序列长度-1；
  3. 对堆顶元素筛运算，重新调整成一个堆。
  4. 重复直到堆为空。

误差

定义\(E(x)=x-\tilde{x}\)为近似值对于真实值的绝对误差；

\(E[f(x)]=f'(x)E(x)\)，对于可微函数成立。

有效数字

已知\(x\)为真实值，而\(\tilde{x}\)为近似值，如果k是下式成立的最大整数，则称\(\tilde{x}\)精确到小数点后第k位。 \[ |x-\tilde{x}|\le \frac{1}{2} \times 10^{-k} \]

前向误差&后向误差

前向误差\(=|\hat{y}-y|\)，后向误差为\(=|F(\hat{y})-x|=|\hat{x}-x|\). \[ \text { cond }=\frac{\mid \text { relative forward error } \mid}{\mid \text { relative backward error } \mid} \] cond<=1表明问题是不敏感的，是良态的；

cond>>1则表明问题是敏感的，是病态的。 \[ \text { cond }=\frac{|[f(\hat{x})-f(x)] / f(x)|}{|(\hat{x}-x) / x|} \approx\left|\frac{f^{\prime}(x) \Delta x / f(x)}{\Delta x / x}\right|=\left|\frac{x f^{\prime}(x)}{f(x)}\right| \] 函数与其反函数，条件数互为倒数。
